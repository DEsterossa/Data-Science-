## Описание проекта

Проект представляет собой Telegram-бота, реализующего Q&A-систему (Question Answering) на основе архитектуры RAG (Retrieval-Augmented Generation). Цель — создать интеллектуального ассистента, способного отвечать на вопросы пользователя строго по содержимому предоставленных им документов.

В отличие от классических поисковых систем, RAG-подход позволяет не просто находить фрагменты текста, а семантически понимать вопрос и генерировать на его основе связный, точный и человекопонятный ответ.

Такие системы являются основой для:
*   Создания корпоративных баз знаний.
*   Автоматизации клиентской поддержки.
*   Разработки персональных ассистентов для анализа больших объемов документации (юридических, технических, научных).

## Навыки и инструменты

**Сферы деятельности:**
LLM-инженерия, RAG, Векторный поиск, API-интеграция, Telegram-боты.

**Ключевые этапы разработки:**
*   **Индексация данных:** Разработан пайплайн для загрузки документов (`.txt`, `.pdf`, `.docx`), их разбиения на семантические чанки и векторизации с помощью `Sentence Transformers`.
*   **Векторное хранение:** В качестве локальной векторной базы данных используется `FAISS` для эффективного хранения и поиска эмбеддингов.
*   **Retrieval (Извлечение):** Настроен механизм извлечения наиболее релевантных чанков на основе семантической близости к вектору вопроса пользователя.
*   **Generation (Генерация):** Сконструирован промпт, который "заземляет" (grounds) большую языковую модель (Google Gemini 2.5 Pro), заставляя ее генерировать ответ исключительно на основе предоставленного контекста.
*   **Оркестрация:** Весь RAG-пайплайн собран и управляется с помощью фреймворка `LangChain`.
*   **Интерфейс:** Реализован многопользовательский интерфейс в виде Telegram-бота с изолированным хранилищем данных для каждого пользователя.

## Выводы и результаты

В ходе проекта была успешно создана и развернута полнофункциональная RAG-система. Бот демонстрирует способность корректно обрабатывать документы различных форматов, находить в них релевантную информацию и генерировать точные, контекстуально-обоснованные ответы.

Ключевым результатом является реализация "безопасного" ответа: система не "галлюцинирует" и честно сообщает, если не может найти ответ в предоставленном документе. Проект подтверждает, что архитектура RAG является эффективным решением для создания современных Q&A-систем, а фреймворк LangChain — мощным инструментом для их оркестрации.

## ⚙️ Установка и запуск

### 1. Клонирование репозитория

```bash
git clone <URL-твоего-репозитория-на-GitHub>
cd <имя-папки-проекта>
```

### 2. Создание и активация виртуального окружения

```bash
# Для Windows
python -m venv venv
.\venv\Scripts\activate

# Для macOS / Linux
python3 -m venv venv
source venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Настройка API-ключей

1.  Создайте файл `.env` в корневой папке проекта.
2.  Получите API-ключ для **Google Gemini** в [Google AI Studio](https://aistudio.google.com/).
3.  Создайте нового бота в Telegram через `@BotFather` и получите его токен.
4.  Добавьте ключи в файл `.env`:

    ```env
    GOOGLE_API_KEY="ВАШ_API_КЛЮЧ_ОТ_GEMINI"
    TELEGRAM_BOT_TOKEN="ВАШ_ТОКЕН_ТЕЛЕГРАМ_БОТА"
    ```

### 5. Запуск бота

```bash
python bot.py
```
После запуска бот будет готов к работе в Telegram.
