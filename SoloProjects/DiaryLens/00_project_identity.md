# DiaryLens / Journal Analyzer — суть проекта

## Главная формулировка

Бот раз в неделю получает дневниковые записи, извлекает из них факты и сигналы, сравнивает их с моими целями и прошлым контекстом, а затем выдаёт не “психологический разбор”, а доказательный недельный отчёт:

- что реально происходило;
- где был прогресс;
- где я буксовал;
- что повторяется;
- где есть самообман или искажения;
- что разумно сделать дальше.

Это не “ИИ, который меня понимает”.

Это **аналитическая система для личного weekly review**.

---

# 1. Главный принцип

Бот должен работать по принципу:

## **от дневника → к структуре → к выводам → к действиям**

Не так:

```text
7 дневниковых файлов → большой промпт → длинный философский ответ
```

А так:

```text
7 дневниковых файлов
→ извлечение фактов за каждый день
→ сбор недельной картины
→ сравнение с целями и прошлым контекстом
→ поиск повторяющихся паттернов
→ итоговый отчёт
→ 1–3 действия на следующую неделю
```

То есть бот не должен сразу “рассуждать”.

Сначала он должен **разложить неделю на данные**.

---

# 2. Что это за система

Journal Analyzer — это система еженедельного анализа личного дневника.

Её задача — не заменять психолога и не давать мотивационные советы, а помогать пользователю трезво видеть:

- что происходило за неделю;
- какие темы повторялись;
- где был реальный прогресс;
- где возникали напряжения;
- какие решения остались незакрытыми;
- что разумно сделать дальше.

Каждую неделю пользователь загружает 7 дневниковых записей. Система сначала извлекает из каждой записи структурированные данные: факты, события, победы, проблемы, эмоции, признаки энергии/усталости, решения и открытые вопросы.

Затем эти данные агрегируются в недельную картину.

После этого система сравнивает неделю с долгосрочными целями пользователя, предыдущими недельными отчётами и сохранёнными паттернами.

Особое внимание уделяется не только эмоциональному фону, но и:

- фактическому фокусу недели;
- прогрессу по важным направлениям;
- рискам распыления;
- тревожной активности;
- противоречиям между целями и действиями;
- незакрытым вопросам.

---

# 3. Главная мысль

Бот должен быть не “эмоциональным анализатором”, а системой еженедельной аналитики жизни:

> он помогает увидеть факты, динамику, повторяющиеся паттерны, соответствие целям и разумный следующий шаг.

Система не должна анализировать ради анализа.

Её цель — помогать пользователю отличать:

- реальные факты от интерпретаций;
- прогресс от имитации движения;
- важное от второстепенного;
- разумное действие от тревожной активности.

---

# 4. Практический вывод

Начинать надо с одного режима — **weekly_full**.

Минимальная первая версия:

```text
7 дневниковых файлов
+ goals.md
+ context_about_me.md
→ weekly_report.md
```

Без Telegram.  
Без сложной памяти.  
Без обучения модели.  
Без разметки.

Сначала нужно добиться, чтобы один weekly report был реально полезнее обычного чата.

Потом уже можно добавлять retrieval, historical patterns и режимы.
